## ⚡Типы и структуры данных



> **Пояснить за изменяемые и неизменяемые типы данных**

- ***Ответ***

  В Python все объекты делятся на **изменяемые** и **неизменяемые** типы данных.

  - **Неизменяемые типы данных:**
Неизменяемые объекты нельзя изменить после их создания. Если вы пытаетесь внести изменения, то фактически создается новый объект, а не изменяется существующий. Примеры неизменяемых типов данных:

  1. **Числа** (`int`, `float`, `complex`)

  ```Python
a = 5
a = 7  # создается новый объект с числом 7
```


  2. **Строки** (`str`)

  ```Python
s = "hello"
s = s + " world"  # создается новая строка "hello world"
```


  3. **Кортежи** (`tuple`)

  ```Python
t = (1, 2, 3)
# t[0] = 10  - вызовет ошибку
```


  4. `frozenset`: неизменяемая версия множества (`set`)

  ```Python
f_set = frozenset([1, 2, 3])
# f_set.add(4)  - вызовет ошибку
```


  - **Изменяемые типы данных:**
Изменяемые объекты можно изменять без создания нового объекта. Примеры изменяемых типов данных:

  5. **Списки** (`list`):

  ```Python
lst = [1, 2, 3]
lst[0] = 10  # изменяется существующий список
```


  6. **Множества** (`set`):

  ```Python
s = {1, 2, 3}
s.add(4)  # множество изменяется
```


  7. **Словари** (`dict`):

  ```Python
d = {'a': 1, 'b': 2}
d['a'] = 10  # изменяется значение ключа 'a'
```


  8. **Массивы** (`bytearray`):

  ```Python
b = bytearray(b"hello")
b[0] = 104  # изменяется первый байт массива
```


  **Как это влияет на поведение:**

  - Для **изменяемых** объектов изменения происходят по ссылке, и все переменные, ссылающиеся на этот объект, увидят изменения.

  - Для **неизменяемых** объектов любое изменение приводит к созданию нового объекта, поэтому другие ссылки на старый объект остаются без изменений.

  ```Python
a = [1, 2, 3]
b = a
a[0] = 10
print(b)  # b также изменится, так как a и b ссылаются на один и тот же список

c = 5
d = c
c = 7
print(d)  # d останется равным 5, так как числа неизменяемы
```




> **Что произойдёт с переменной B, если B = A, а затем A += 1?**

- ***Ответ***

  Поведение зависит от типа объекта — изменяемый/неизменяемый. 

  **Пусть A — неизменяемый объект и мы делаем:**

  ```Python
A = 1
B = A
A += 1
```


  Что здесь произойдёт:

  - `A = 1` → `A` указывает на объект `1`

  - `B = A` → `B` тоже указывает на `1`

  - `A += 1` → это **создаёт новый объект `2`** и `A` теперь ссылается на него

  - `B` остаётся указывать на старый `1`

  Итог: `A = 2`, `B = 1`

  **Теперь пусть А — изменяемый объект и мы делаем:**

  ```Python
A = [1, 2]
B = A
A.append(3)
```


  Что будет тут:

  - `A = [1, 2]`, `B = A` — теперь `A` и `B` указывают на один и тот же список

  - `A.append(3)` — список **А** становится `[1, 2, 3]`, а так как **B** указывает на тот же список, то он тоже увидит изменение

  Итог: `A = [1, 2, 3]` и `B = [1, 2, 3]` — то есть изменятся **ОБА.**



> **Как работает словарь в Python? Какие сложности операций и как разрешаются коллизии?**

- ***Ответ***

  `dict` — неупорядоченная коллекция произвольных объектов с доступом по ключу (реализован на основе хэш-таблица). Состоит из ключа - значения.
**Ключ**: Любой хэшируемый объект (строки, числа, кортежи и т.д.). Ключ должен быть неизменяемым, чтобы хэш не менялся на протяжении жизни объекта в словаре.
**Значение**: Любое значение, которое ассоциируется с ключом, может быть изменяемым или неизменяемым.

  - Сложности операций: Большая часть O(1), перебор O(n)

  - Работа словаря:
Когда мы вставляем пару "ключ-значение" в словарь, Python выполняет следующие шаги:

  1. **Хэш-функция**: Для каждого ключа вызывается хэш-функция, которая возвращает хэш-значение (целое число). В Python это делается с помощью функции `hash()`. Хэш-функция предназначена для того, чтобы равномерно распределять ключи по хэш-таблице, минимизируя вероятность коллизий.

  2. **Расчет индекса**: Полученное хэш-значение преобразуется в индекс внутри массива хэш-таблицы, например, через операцию деления по модулю (`hash(key) % size_of_table`). Это определяет ячейку, куда будет помещен элемент.

  3. **Помещение элемента**: Ключ и значение сохраняются в ячейку хэш-таблицы по рассчитанному индексу. Если эта ячейка уже занята другим элементом (коллизия), применяется стратегия разрешения коллизий.

  - **Разрешение коллизии:**
В Python используется метод **открытой адресации** для разрешения коллизий. Если наш ключ-значение попадает в ячейку, где уже все занято, то пользуемся механизмом пробинга, чтобы поместить в свободную ячейку. В современном Python это квадратичный пробинг: «quadratic probing» с perturbation (смещение + случайное блуждание). Квадратичный пробинг работает лучше, чем обычный линейный (когда мы каждый раз двигаемся на i+1 и ищем свободную ячейку), 

  - Надо также учитывать, что словарь при заполнении на примерно 2/3 расширяется.



> **Как работает квадратичный пробинг в хеш-таблицах?**

- ***Ответ***

  В современном Python при обработке коллизий используется именно этот механизм пробинга. Вместо того чтобы искать следующую ячейку (`i + 1`, `i + 2`, как при линейном пробинге), мы идём таким образом:

  ```Python
i + 1^2, i + 2^2, i + 3^2, i + 4^2, ...
```


  То есть мы расширяем поиск нелинейно, "перепрыгивая" ячейки.

  **Пример:**

  Пусть у нас 8 ячеек (индексы 0..7) и ключ `key1` с хешем → `i = 3`, но слот `3` уже занят.

  **Quadratic probing** пойдёт по:

  - 3 + 1^2 = 4

  - 3 + 2^2 = 7

  - 3 + 3^2 = 12 → 12 % 8 = 4 ← снова занято!

  - 3 + 4^2 = 19 → 19 % 8 = 3 ← снова...

  - и т.д.

  (на практике берётся маска `& (size - 1)`, а не `%` — чтобы быстрее было, но идея та же). 

  **Зачем так сложно?**

  - Линейный пробинг приводит к кластеризации: занятые слоты начинают сбиваться в кучи → хуже производительность

  - Quadratic probing разрывает такие кучи и лучше распределяет поиски → меньше коллизий → быстрее работа



> **Как устроен set в Питоне, чем отличается set от dict?**

- ***Ответ***

  set реализован как хэш-таблица, аналогично dict, но с одной ключевой особенностью:

  - set хранит только ключи (уникальные элементы);

  - Значений у set нет — только "ключи", ради которых ведётся вся структура.



> **Какова сложность операций вставки, удаления, поиска в dict и в set?**

- ***Ответ***

  У всех этих операциях в словарях и множествах сложность O(1). Обе структуры реализованы как хэш-таблицы, поэтому операции происходят примерно так:

  1. Вычисляется `hash(key)`

  2. Определяется "слот" в таблице

  3. Элемент читается или обновляется напрямую

  Примеры:

  ```Python
d = {}
d['a'] = 1        # O(1)
print('a' in d)   # O(1)
del d['a']        # O(1)

s = set()
s.add(10)         # O(1)
print(10 in s)    # O(1)
s.remove(10)      # O(1)
```




> **Как устроен список в Питоне? Какая сложность у доступа к элементу списка по индексу?**

- ***Ответ***

  **— База**

  `list` — это **динамический массив**. Внутри — **непрерывный блок памяти** (array), где хранятся **указатели** (ссылки) на объекты, а не сами объекты. То есть `list` — это массив **PyObject** (адресов объектов в памяти).

  Пример:

  ```Plain Text
arr = [ 10, "abc", [1,2] ]
```


  В памяти:

  ```Plain Text
arr  → [ ptr1, ptr2, ptr3 ]
ptr1 → объект int(10)
ptr2 → объект str("abc")
ptr3 → объект list([1,2])
```


  

  **— Размер и вместимость**

  У списка есть:

  - **size** (len) — сколько элементов сейчас в списке;

  - **capacity** — сколько выделено места (может быть больше `len`).

  Когда делаем `append()`:

  - если есть свободное место, просто добавляется указатель;

  - если места нет, выделяется новый блок большего размера (**обычно capacity увеличивается в 1.5–2 раза**), и все указатели копируются туда (**релокация**).

  Поэтому `append()` работает за амортизированное `O(1)`.

  

  **— Срезы и доступ по индексу**

  - Так как данные хранятся в непрерывном массиве указателей,
доступ по индексу (`arr[i]`) выполняется за **O(1)**.

  - Срез (`arr[a:b]`) создаёт **новый список** с копиями указателей (не самих объектов).

  Пример:

  ```Plain Text
nums = [10, 20, 30, 40]
part = nums[1:3]
print(part)  # [20, 30]

nums[1] = 99
print(part)  # [20, 30] (не меняется, потому что новый список)
```


  

  **— Изменяемость**

  - `list` в Python — **изменяемый** тип.

  - Можно изменять элементы, добавлять, удалять.

  - При этом сами объекты могут быть как изменяемыми (`list`, `dict`), так и неизменяемыми (`int`, `str`, `tuple`).

  

  **— Важные операции и их сложность**

  |Операция|Средняя сложность|
|-|-|
|`arr[i]` (индекс)|O(1)|
|`len(arr)`|O(1)|
|`arr.append(x)`|O(1) амортизированно|
|`arr.pop()` (с конца)|O(1)|
|`arr.insert(0, x)`|O(n)|
|`arr.pop(0)`|O(n)|
|`arr[a:b]` (срез)|O(k), где k = длина среза|

  **— Особенности**

  - Список может хранить объекты **разного типа** (в отличие от массивов C/Java)

  - На самом деле хранится массив **указателей**

  - При этом на каждый элемент есть **одна дополнительная разыменовка** (доступ медленнее, чем в C++ `vector<int>`)



> **Можно ли изменить список, вложенный в tuple?**

- ***Ответ***

  Да, список, вложенный в tuple, можно изменить, потому что tuple не делает содержимое неизменяемым, а лишь запрещает менять саму структуру кортежа (например, нельзя заменить элемент). Пример:

  ```Python
t = ([1, 2, 3], 42)
t[0].append(4) # так можно, ошибки не будет
t[0] = [9, 9, 9]  # а вот так нельзя, будет TypeError
```




> **Какова сложность вставки и удаления элементов в разных частях списка?**

- ***Ответ***

  Список в Python реализован как динамический массив (array), то есть все элементы хранятся в непрерывной области памяти. 

  **Вставка и удаление элемента в начале — это O(n):**

  ```Python
lst = [10, 20, 30]
lst.insert(0, 5)  # => [5, 10, 20, 30]
```


  Чтобы вставить 5 в начало, Python:

  - Выделяет место в памяти, если нужно

  - Сдвигает все элементы вправо (копирует)

  - Вставляет 5 в `index=0`

  Сдвиг n элементов = O(n). С удалением точно так же, только элементы двигаются влево.

  **Вставка и удаление элемента в конце — это O(1):**

  Тут элемент просто добавляется в следующий свободный слот памяти. Если места нет — список расширяется (перевыделяется с запасом), но это редкая операция, поэтому амортизированная сложность остаётся O(1).



> **Почему список внутри функции может не изменять оригинальный список?**

- ***Ответ***

  Список внутри функции может не изменить оригинал, если внутри функции мы создаём новый список, а не модифицируем существующий. Важно понимать, что списки передаются по ссылке (на объект), а не копируются при передаче в функцию. Но если мы переприсваиваем параметр (например, `lst = [...]`), мы создаём новый локальный список, не затрагивая оригинал.

  Пример, где оригинальный список изменится:

  ```Python
def modify(lst):
    lst.append(4)

a = [1, 2, 3]
modify(a)
print(a)  # [1, 2, 3, 4]
```


  Пример, где оригинальный список не изменится:

  ```Python
def modify(lst):
    lst = [4, 5, 6]  # создаётся новый список, ссылка теряется

a = [1, 2, 3]
modify(a)
print(a)  # [1, 2, 3]
```

## ⚡Общие вопросы по Python



> **Что такое `*args` и `**kwargs`?**

- ***Ответ***

  `*args` — собирает **позиционные** аргументы в кортеж,

  `**kwargs` — собирает **именованные** аргументы в словарь.

  ```Python
def f(*args, **kwargs): 
    print(args, kwargs)

f(1, 2, a=3, b=4)
# (1, 2) {'a': 3, 'b': 4}
```




> **Как работает менеджер контекста в Python? Приведи примеры использования.**

- ***Ответ***

  **Менеджер контекста** — это объект, который управляет выполнением блока кода, обрабатывая действия до и после выполнения блока с использованием ключевого слова `with`. Он гарантирует, что определенные ресурсы, такие как файлы, сетевые соединения или блокировки, будут корректно и автоматически освобождены, даже если в блоке кода происходит ошибка.

  
**Менеджеры контекста реализуются с помощью магических методов**
`__enter__()` и `__exit__()`.

  - **`__enter__()`**: Этот метод выполняется, когда вы входите в контекст (при начале выполнения блока кода после ключевого слова `with`). Он может вернуть объект, который будет доступен в блоке кода.

  - **`__exit__()`**: Этот метод выполняется, когда вы покидаете контекст (после завершения выполнения блока кода или при возникновении исключения). Он обычно используется для очистки или освобождения ресурсов, таких как закрытие файла или освобождение соединения.
Примеры:

  ```Python
# Открываем файл и автоматически закрываем его после выполнения блока
with open('example.txt', 'r') as file:
    content = file.read()
    print(content)
# Файл автоматически закроется после выхода из блока

# Подключения к базам данных
with DatabaseConnection() as conn: # Работа с базой данных

# Блокировки потоков
from threading import Lock
lock = Lock()

with lock:
    # Код, который требует синхронизации потоков
```




> **Что такое итерируемые объекты, итераторы, генераторы? Чем они отличаются?**

- ***Ответ***

  - **Итерируемый (`iterable`) объект** — это объект, у которого можно вызвать `iter()`, чтобы получить итератор. **Примеры итерируемых объектов:** списки, строки, кортежи, словари, множества, объекты `range`, объекты `enumerate`, `zip`, `map`, `filter`, а также любой объект с методом `__iter__()`

  - **Итератор** — это объект, который реализует методы `__iter__()` и `__next__()`. Итератор по одному возвращает элементы последовательности. При этом каждый следующий элемент запрашивается при помощи `next()`, и итератор запоминает, на каком элементе остановился. Когда элементы заканчиваются, он вызывает исключение `StopIteration`

  - **Генератор** — это специальная функция, которая использует ключевое слово `yield` для создания последовательности значений. В отличие от итераторов, генераторы более удобны для написания и не требуют создания отдельного класса. Каждое выполнение `yield` возвращает значение и "замораживает" состояние функции. При следующем вызове функция продолжает выполнение с того места, где остановилась

  **Пример итератора:**

  ```Python
class MyIterator:
    def __init__(self, max_value):
        self.current = 0
        self.max_value = max_value

    def __iter__(self):
        return self

    def __next__(self):
        if self.current < self.max_value:
            result = self.current
            self.current += 1
            return result
        else:
            raise StopIteration

# Используем итератор
iterator = MyIterator(5)
for value in iterator:
    print(value)
```


  **Пример генератора:**

  ```Python
def my_generator(max_value):
    current = 0
    while current < max_value:
        yield current
        current += 1

# Используем генератор
for value in my_generator(5):
    print(value)
```




> **Что такое декораторы и как они работают?**

- ***Ответ***

  **Декораторы** в Python — это функции, которые позволяют модифицировать или расширять поведение других функций или методов **без изменения их исходного кода**. Они позволяют "оборачивать" одну функцию в другую, добавляя дополнительную логику перед или после выполнения основной функции.

  - **Как работают декораторы?**
Декораторы принимают функцию в качестве аргумента и возвращают новую функцию, которая обычно вызывает оригинальную функцию и добавляет к ней какую-то логику. Это делается с помощью специального синтаксиса **`@декоратор`**, который размещается перед определением функции.

  ```Python
def log_decorator(func):
    def wrapper(*args, **kwargs):
        print(f"Вызов функции {func.__name__} с аргументами {args} и {kwargs}")
        result = func(*args, **kwargs)
        print(f"Результат функции {func.__name__}: {result}")
        return result
    return wrapper

@log_decorator
def add(a, b):
    return a + b

# Вызов функции
add(2, 3)

# Результат - очень просто реализовали логирование функции
Вызов функции add с аргументами (2, 3) и {}
Результат функции add: 5
```


  Стоит отметить несколько моментов:

  1. Можно использовать несколько декораторов друг за другом. Выполняться они по порядку с самого 'верхнего'.

  2. Декоратору можно передать аргумент

  ```Python
def requires_auth(func):
    def wrapper(*args, **kwargs):
        user = kwargs.get('user')
        if user == 'admin':
            return func(*args, **kwargs)
        else:
            print("Доступ запрещен")
    return wrapper

@requires_auth
def view_dashboard(user):
    print(f"Добро пожаловать, {user}!")

view_dashboard(user='admin')  # Добро пожаловать, admin!
view_dashboard(user='guest')  # Доступ запрещен
```




> **Что такое лямбда-функции и в чем их отличие от обычных функций?**

- ***Ответ***

  **Лямбда-функции** (или **анонимные функции**) — это небольшие, одноразовые функции, которые могут быть созданы с помощью ключевого слова `lambda`. Лямбда-функции позволяют писать краткие функции без использования ключевого слова `def` и без необходимости давать им имя.
Основные характеристики лямбда-функций:

  - Они могут иметь любое количество аргументов, но только **одно выражение**.

  - В отличие от обычных функций, они **не имеют имени** (поэтому их называют анонимными).

  - Лямбда-функции часто используются там, где требуется небольшая, одноразовая функция, например, в таких функциях как `map()`, `filter()` и других.

  ```Python
# Обычная лямбда-функция, которая складывает два числа
add = lambda x, y: x + y

print(add(3, 5))  # Вывод: 8

def add(x, y):
    return x + y

print(add(3, 5))  # Вывод: 8

# Сортировка списка кортежей по второму элементу
data = [(1, 2), (3, 1), (5, 0)]
sorted_data = sorted(data, key=lambda x: x[1])
print(sorted_data)  # Вывод: [(5, 0), (3, 1), (1, 2)]
```

> **Как работает сборщик мусора и управление памятью в Python?**

- ***Ответ***

  Python автоматически управляет памятью через систему **управления памятью** и **сборщика мусора**. Эти механизмы освобождают память, занятую объектами, которые больше не используются, предотвращая утечки памяти и делая программирование более удобным.
Основные компоненты управления памятью в Python:

  - **Подсчёт ссылок (Reference Counting)**:

    - В Python каждый объект имеет счетчик ссылок — **сколько раз на объект ссылаются переменные**. Этот счётчик увеличивается при создании новой ссылки и уменьшается при удалении ссылки.

    - Когда счётчик ссылок объекта становится равен нулю, Python автоматически освобождает память, занятую этим объектом.

    ```Python
a = []  # Счётчик ссылок на список: 1
b = a   # Счётчик ссылок на список: 2
del a   # Счётчик ссылок на список: 1
del b   # Счётчик ссылок на список: 0, объект удалён

```


  - **Сборка циклических ссылок (Garbage Collector)**:

    - Подсчёт ссылок не справляется с **циклическими ссылками** — когда два объекта ссылаются друг на друга, и их ссылки никогда не обнуляются.

    - Python использует **алгоритм циклического обнаружения** для сбора таких объектов. Это происходит в фоновом режиме сборщиком мусора, который проверяет объекты на наличие циклических ссылок и удаляет их, если они больше не используются.

  - **Поколения объектов (Generational Garbage Collection)**:

    - Python разделяет объекты на **3 поколения** (генерации) на основе их "возраста" в памяти. Сборщик мусора чаще проверяет молодые объекты и реже — старые.

    - Новые объекты помещаются в **первое поколение**. Объекты, которые переживают несколько циклов сборки мусора, перемещаются в следующее поколение. Чем старше объект, тем реже его проверяют.

> **Что такое магические методы в Python и как их использовать?
В чем разница между `__str__` и `__repr__`?**

- ***Ответ***

  **Магические методы** (или **специальные методы**) — это методы, которые начинаются и заканчиваются двумя символами подчеркивания (`__`). Эти методы позволяют **определять поведение объектов для встроенных функций и операторов.** Магические методы автоматически вызываются Python при выполнении определённых действий, таких как сложение объектов, сравнение, преобразование в строку и т.д. Это основа для реализации **перегрузки операторов** и предоставляют возможность управлять поведением объектов в Python. Например, вы можете контролировать, как объект будет реагировать на операторы сложения, умножения, сравнения или даже преобразования в строку.
Пример использования и разница между `__str__` и  `__repr__`.

  ```Python
class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    # __str__ для удобного вывода пользователю
    def __str__(self):
        return f"Person(name={self.name}, age={self.age})"

    # __repr__ для отладки и разработки
    def __repr__(self):
        return f"Person('{self.name}', {self.age})"

p = Person("Alice", 30)

# __str__ вызывается при использовании print()
print(p)  # Вывод: Person(name=Alice, age=30)

# __repr__ вызывается в интерактивной оболочке или через repr()
print(repr(p))  # Вывод: Person('Alice', 30)
```


  1. **Создание и уничтожение объектов**:

  - **`__init__(self, ...)`**: Инициализация объекта.

  - **`__del__(self)`**: Уничтожение объекта.

  2. **Строковое представление**:

  - **`__str__(self)`**: Строковое представление для пользователя (print, str).

  - **`__repr__(self)`**: Официальное представление для отладки (repr).

  3. **Арифметические операции**:

  - **`__add__(self, other)`**: Оператор сложения `+`.

  - **`__sub__(self, other)`**: Оператор вычитания `-`.

  4. **Сравнение объектов**:

  - **`__eq__(self, other)`**: Оператор равенства `==`.

  - **`__lt__(self, other)`**: Оператор меньше `<`.

  5. **Итерации**:

  - **`__iter__(self)`**: Возвращает итератор (используется в циклах).

  - **`__next__(self)`**: Возвращает следующий элемент итератора.

  6. **Контекстные менеджеры**:

  - **`__enter__(self)`**: Вход в контекст (`with`).

  - **`__exit__(self, exc_type, exc_val, exc_tb)`**: Выход из контекста.

  7. **Операции с элементами (индексы)**:

  - **`__getitem__(self, key)`**: Доступ по индексу (`[]`).

  - **`__setitem__(self, key, value)`**: Присваивание по индексу (`[]`).

  8. **Вызов объекта**:

  - **`__call__(self, ...)`**: Делает объект вызываемым как функцию.



> **Для чего используется `super`?**

- ***Ответ***

  **`super()`** — это встроенная функция в Python, которая используется для вызова методов родительского (или суперкласса) в классе-наследнике. Она позволяет получить доступ к методам и атрибутам родительского класса из дочернего класса.

  - **Вызов родительских методов**: Она позволяет вызывать методы базового класса внутри методов дочернего класса, не указывая имя базового класса напрямую. В том числе, чтобы не дублировать код и обеспечить правильное поведение при наследование.

  - **Поддержка множественного наследования**: `super()` корректно работает с множественным наследованием и позволяет обращаться к методам родительских классов в правильном порядке (согласно порядку разрешения методов - **MRO**).

  ```Python
class A:
    def greet(self):
        print("Class A")

class B(A):
    def greet(self):
        print("Class B")
        super().greet()

class C(A):
    def greet(self):
        print("Class C")
        super().greet()

class D(B, C):
    def greet(self):
        print("Class D")
        super().greet()

# Пример использования
d = D()
d.greet()

# Вывод
Class D
Class B
Class C
Class A
```




> **Что такое декораторы `classmethod` и `staticmethod`?**

- ***Ответ***

  **`classmethod`** и  **`staticmethod`** — это два декоратора, которые позволяют определить методы в классе.

  1. `@classmethod`

  - **`@classmethod`** используется для создания методов, которые привязаны к самому **классу**, а не к экземпляру класса.

  - Первый аргумент метода с декоратором `@classmethod` должен быть **`cls`** (ссылка на класс), а не **`self`** (ссылка на объект экземпляра класса).

  - Такие методы могут взаимодействовать с классом, но не требуют создания экземпляра.

  ```Python
class MyClass:
    class_variable = 0

    @classmethod
    def increment_class_variable(cls):
        cls.class_variable += 1

# Вызов метода без создания экземпляра
MyClass.increment_class_variable()
print(MyClass.class_variable)  # Вывод: 1
```


  2. `@staticmethod`

  - **`@staticmethod`** используется для создания методов, которые не зависят ни от класса, ни от экземпляра класса. Такие методы не принимают ни аргумент `cls`, ни `self`.

  - Это просто функции, которые находятся внутри класса, но не имеют доступа к его состоянию или атрибутам.

  ```Python
class MyClass:
    @staticmethod
    def multiply(a, b):
        return a * b

# Вызов метода без создания экземпляра
result = MyClass.multiply(3, 4)
print(result)  # Вывод: 12
```

> **Что такое инкапсуляция и как её реализуют в Python?**

- ***Ответ***

  Инкапсуляция — это принцип ООП, при котором внутреннее состояние объекта скрывается от внешнего мира, и доступ к нему осуществляется только через методы (геттеры/сеттеры). В Python нет строгого механизма доступа (как `private`, `protected` в Java/C++), но есть соглашения**:**

  - `var` — публичный уровень доступа к атрибуту, можно свободно использовать

  - `_var` — protected атрибут, не использовать вне класса/подклассов

  - `__var` — private атрибут, это называется искажением имени (name mangling) — Python автоматически меняет имя переменной, добавляя к нему имя класса. Это нужно, чтобы "спрятать" переменную от внешнего доступа и защитить её от случайного использования или переопределения в дочерних классах.

  Пример:

  ```Python
class BankAccount:
    def __init__(self, owner, balance):
        self.owner = owner              # публичный
        self._balance = balance         # protected (по соглашению)
        self.__transaction_log = []     # private (name mangling)
```




## ⚡**Многопоточность и многопроцессность**



> **Чем отличаются потоки и процессы в Python?
Как Python работает с многопоточностью и многопроцессорностью?**

- ***Ответ***

  - **Процессы:**
Процесс — это независимая единица выполнения кода с собственным выделенным пространством памяти. Запуск нескольких процессов позволяет Python выполнять задачи параллельно, так как каждый процесс работает независимо и не делит память с другими процессами.
Многопроцессорность в Python реализуется при помощи модуля
`multiprocessing`, который создает отдельные процессы. Это помогает обходить GIL (Global Interpreter Lock), который ограничивает одновременное выполнение кода в одном потоке.

  - **Потоки:**
Поток (или нить) — это меньшая единица выполнения, которая выполняется внутри одного процесса. Потоки одного процесса разделяют его память и ресурсы, что упрощает передачу данных между ними, но создает риски для конкурентного доступа к общим ресурсам.
В Python многопоточность реализуется с помощью модуля `threading`. Однако, из-за GIL только один поток может исполнять Python-код в каждый конкретный момент времени, что ограничивает реальную параллельность выполнения.



> **Что такое GIL (Global Interpreter Lock) и как он влияет на многопоточность?**

- ***Ответ***

  - GIL — это механизм Python, который предотвращает одновременное выполнение нескольких потоков на уровне интерпретатора. GIL делает многопоточность менее эффективной для задач с интенсивной нагрузкой на процессор, так как потоки должны поочередно ждать, чтобы получить доступ к интерпретатору.

  - Из-за GIL многопоточность в Python больше подходит для задач, ограниченных ожиданием ввода-вывода (например, работа с сетью или файлами), а для вычислительно интенсивных задач (например, математика, обработка изображений) часто выбирают многопроцессорность.



> **Как происходит работа с памятью при `multiprocessing`?**

- ***Ответ***

  При использовании модуля **`multiprocessing`** для параллельного выполнения задач, каждый процесс в Python имеет **собственное адресное пространство**. Это означает, что процессы **не разделяют память**, как это происходит в случае с потоками, использующими один и тот же процесс (где возникает проблема с GIL).

  Каждый процесс в `multiprocessing` является **отдельным экземпляром интерпретатора Python**, со своей собственной памятью, поэтому работа с памятью в `multiprocessing` имеет свои особенности.

  Основные моменты работы с памятью при использовании `multiprocessing`

  1. **Изоляция памяти**:

    - Когда создается новый процесс с помощью `multiprocessing`, он получает **полную копию** адресного пространства родительского процесса, но после создания **любой объект в дочернем процессе независим** от объекта в родительском процессе.

    - Это означает, что если объект изменен в дочернем процессе, изменения **не будут видны** в родительском процессе и наоборот.

    Пример:

    ```Python
from multiprocessing import Process

def modify_list(my_list):
    my_list.append(100)
    print("Inside process:", my_list)

my_list = [1, 2, 3]
p = Process(target=modify_list, args=(my_list,))
p.start()
p.join()

print("Outside process:", my_list)
```


    Вывод:

    ```Plain Text
Inside process: [1, 2, 3, 100]
Outside process: [1, 2, 3]
```


    В данном примере видно, что изменения в списке внутри дочернего процесса не влияют на список в родительском процессе, так как процессы не разделяют память.

  2. **Общая память (Shared Memory)**:

    - Чтобы организовать обмен данными между процессами, используются **специальные механизмы**, такие как **очереди** (`Queue`), **каналы** (`Pipe`) и **общая память** (`Value` и `Array`).

    Эти механизмы позволяют процессам обмениваться данными **безопасно и эффективно**, сохраняя изоляцию памяти.

    Пример с использованием **`Queue`** для обмена данными между процессами:

    ```Python
from multiprocessing import Process, Queue

def worker(q):
    q.put([1, 2, 3])

q = Queue()
p = Process(target=worker, args=(q,))
p.start()
print(q.get())  # Получаем данные от дочернего процесса
p.join()
```


    - **Очереди (Queue)** и **каналы (Pipe)** используются для передачи данных между процессами, поддерживая изоляцию памяти, но позволяя передавать объекты между ними.

  3. **Объекты в общей памяти (shared memory)**:

    - Модуль `multiprocessing` предоставляет объекты **`Value`** и **`Array`** для создания переменных и массивов в общей памяти, к которым могут обращаться несколько процессов.

    - Эти объекты используются, когда нужно поделиться небольшими данными (например, числами, массивами) между процессами без копирования.

    Пример с **`Array`**:

    ```Python
from multiprocessing import Process, Array

def modify_array(shared_array):
    for i in range(len(shared_array)):
        shared_array[i] += 1

shared_array = Array('i', [1, 2, 3])
p = Process(target=modify_array, args=(shared_array,))
p.start()
p.join()

print(shared_array[:])  # Вывод: [2, 3, 4]
```


    Здесь **`Array`** предоставляет доступ к разделяемому массиву, и изменения, сделанные в одном процессе, будут видны в другом процессе.

  4. **Копирование данных при передаче**:

    - Когда данные передаются между процессами (например, через **`Queue`** или **`Pipe`**), они **копируются**. Это увеличивает время передачи данных и использует больше памяти, чем в случае потоков, где данные передаются по ссылке.

    - Поэтому для передачи больших объемов данных между процессами использование механизмов вроде общей памяти предпочтительнее.

  5. **Блокировки и синхронизация**:

    - Для безопасного доступа к общей памяти при `multiprocessing` могут потребоваться **блокировки** (`Lock`), **события** (`Event`), **семафоры** (`Semaphore`), чтобы предотвратить одновременный доступ к разделяемым ресурсам несколькими процессами.

    Пример с **`Lock`**:

    ```Python
from multiprocessing import Process, Lock

def worker(lock, num):
    with lock:
        print(f"Process {num} is running")

lock = Lock()
processes = [Process(target=worker, args=(lock, i)) for i in range(5)]

for p in processes:
    p.start()

for p in processes:
    p.join()
```


    - Здесь `Lock` гарантирует, что процессы будут поочерёдно выводить свои сообщения, предотвращая гонки состояний.
Основные механизмы для работы с памятью в `multiprocessing`

  - **Очереди (Queue)**: Позволяют передавать объекты между процессами через специальную очередь.

  - **Каналы (Pipe)**: Прямой двусторонний обмен данными между двумя процессами.

  - **`Value` и `Array`**: Объекты для хранения данных в общей памяти, доступных всем процессам.

  - **Блокировки (Lock)**: Используются для синхронизации доступа к общим ресурсам, предотвращая одновременный доступ.

  - **Менеджеры (Managers)**: Позволяют создавать серверы объектов для обмена сложными объектами (например, списками или словарями) между процессами.

> **Какие ограничения есть при передаче данных между процессами?**

- ***Ответ***

  Процессы в Python (в отличие от потоков) не разделяют память. Поэтому чтобы передавать данные между процессами, они должны быть сериализуемыми (picklable) — т.е. поддерживать `pickle`, иначе Python не сможет "упаковать" объект и передать его через канал (`Queue`, `Pipe`, `Manager`). 

  **Что нельзя передавать между процессами напрямую?**

  Непиклируемые объекты:

  - Открытые файлы / сокеты

  - Лямбда-функции (`lambda`)

  - Объекты, содержащие непиклируемые поля (например, потоки, дескрипторы)

  - Методы экземпляров (если класс не поддерживает `__getstate__`)

  **Что можно передавать?**

  - Простые типы: `int`, `str`, `list`, `dict`, `tuple`

  - NumPy-массивы (если явно сериализуются)

  - Объекты, которые реализуют `__getstate__` и `__setstate__`

  - Всё, что `pickle.dumps()` работает без ошибки

  **Как проверить, что можно передать объект?**

  ```Python
import pickle
pickle.dumps(my_obj)  # если здесь ошибка — объект нельзя передать в другой процесс
```
